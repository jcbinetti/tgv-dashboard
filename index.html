Kategorie-Logik L1 / L2 / L3

Das System klassifiziert jede Transaktion auf 3 Ebenen. L1 ist fix (12 Werte), L2 ist pro L1 vordefiniert + erweiterbar, L3 ist frei für Sonderfälle.

VersicherungAlle VersicherungsprämienKrankenversicherung, KFZ, Haftpflicht, Lebensversicherung…

Workflow-Erklärung

WorkflowWas macht er?TriggerStatus
WF A v9.x Hauptworkflow: liest Emails (IMAP), erkennt CSVs und PDFs, importiert Transaktionen in Supabase, klassifiziert mit KB + Claude AI, lädt Dokumente zu Google Drive hoch Alle 15 Min automatisch 🟢 Aktiv
WF B Google Drive Scanner: scannt Drive-Ordner auf neue Dateien Alle 15 Min automatisch 🟢 Aktiv
WF C Claude AI Review: nimmt alle ai_reviewed/pending Txn, schickt sie an Claude API zur Klassifizierung, schreibt Ergebnis in ai_suggestion + Spalten direkt Täglich 08:00 UTC 🔴 API-Limit bis 01.05
WF E Buchhalter-Paket: sendet monatlich eine Zusammenfassung per Email an Buchhalter 1. des Monats 08:00 🟡 Prüfen ob aktiv
WF F Fehlende Beträge: prüft Dokumente ohne Betrag — erst sinnvoll wenn Dokumentenimport vollständig — ⚪ Nicht importiert — vorerst nicht nötig
CSV importieren: Bank-CSV per Email an factures@convis.fr schicken → WF A erkennt es automatisch → Txn landen in Supabase → erscheinen im Review-Tab.

Neue L2-Kategorie: Im Review-Tab L2-Dropdown → "+ Neu…" → Eingabe → automatisch in Supabase gespeichert → dauerhaft verfügbar.

Vorschuss markieren: Im Review-Tab, Feld "L3 / Vorschuss" → "Vorschuss C3" eingeben → OK klicken → erscheint in Konten-Tab → Vorschuss-Liste.
